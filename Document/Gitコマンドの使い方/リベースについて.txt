リベース -> あるブランチを作業中に他の人がmainブランチを更新してしまった(分岐)ケースで、あたかも最新のmainブランチからブランチしたかのようにするコマンド
　自分だけの開発でよく使う(履歴が汚くなってしまったときなど)
　複数人の開発では気軽にリベースはできない

例:
　git checkout br-b
　git rebase main

git reflog -> リベースする前の過去の状態を見ることができる(ガベージコレクションで自動削除されてしまうまでは)
git rebase -i 起点となるコミット -> コミット順を並び替えたり、コミットメッセージを変更したり、対話的にリベースできる
git commit --amend -> HEADが指しているコミットを編集。git addを更に追加したり、削除したり、コミットメッセージを編集できる。

git rebase -i
　pick -> 現在のコミットを採用
　edit -> 現在のコミットを採用するが、修正のため中断する(amendできる)
　drop ->コミットを採用しない
