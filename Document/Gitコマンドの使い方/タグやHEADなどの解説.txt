コミットオブジェクト -> その時点のワークツリー(ファイル、フォルダ)全体の情報(スナップショット)
　差分は持っていない（gitは差分を保存して管理していると思われがちだが、各コミットでスナップショットを保存して管理している）
　それぞれのオブジェクトはSHA1でハッシュされたIDを持っている

ブランチ -> コミットについたラベル(ポインタ)
　ブランチに対してコミットすると、そのコミットにブランチが追従する（最新コミットに追従する）

HEAD -> 現在作業中のコミットを指す特別なラベル
　git diff（引数なし）などの各種コマンドが暗黙に利用する
　通常は、ブランチにくっついて一緒に動く
　Detached HEADという場合は、ブランチに紐付かないで特定のコミットを指した状態（あるコミットの調査をしたいときなどに使う）

タグ -> コミットについたラベル（v1.0みたいなものをつける）
　ブランチと違い、最新コミットには追従しない
